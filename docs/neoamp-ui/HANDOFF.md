# NeoAmp — Winamp Player UI · Handoff

Handoff for the **Winamp-style player UI** (separate workstream from the Alchemy/
Butterchurn *visualizer presets*, which CLAUDE.md and `docs/alchemy-*` cover). This
is the floating windowed player overlaid on YouTube Music, **now with a real
equalizer**.

Last updated: 2026-06-19. Repo state: on `main`.

## 1. What it is + file map

A Manifest V3 player drawn over music.youtube.com, themed by real `.wsz` skins.
**The audio now runs in an offscreen document** (see §3) — a fundamental change from
the old getDisplayMedia-in-the-content-script design.

| File | Role |
| --- | --- |
| `sw.js` | **Service worker (new).** Owns the gesture-gated `chrome.tabCapture` — the only way to start capture. Toolbar-icon click / right-click menu / `Ctrl+Cmd+Shift+E` all call `toggleCapture`. Creates the offscreen doc, and is the **relay hub**: EQ changes (content→offscreen), FFT frames (offscreen→content), lifecycle (start/stop→content). |
| `offscreen.html` / `offscreen.js` | **Offscreen audio engine (new).** Consumes a tabCapture streamId via `getUserMedia({audio:{mandatory:{chromeMediaSource:'tab',chromeMediaSourceId}}})`, builds `src → preamp → 10 peaking biquads → StereoPanner(balance) → master → destination` (audible, EQ'd), taps an analyser, and ships FFT back. Flat by default = transparent. |
| `content.js` | Backend on `window.NeoAmp`. **No longer captures audio** — it's now SW-driven: receives `lifecycle`/`fft` messages, emits the same `start`/`stop`/`audio` events the UI already consumed. Still scrapes YTM (track/queue/search/home/like), drives playback, and owns `eqState` (the UI's source of truth) which it relays to the offscreen + persists. |
| `winamp.js` | Window manager + DOM windows (Main, EQ, Viz iframe, Playlist, Library, Now-Playing). Skin picker. **EQ window faders → `NA.control.setEqBand/setPreamp/setEqEnabled` → relayed live to the offscreen engine.** In-page ◢◤ launcher is **disabled** (code kept in `ensureLauncher`). |
| `wsz.js` | Real Winamp 2 `.wsz` engine (blits BMP sprites to canvas). `mountMain`/`mountEq` (faders wired to hooks), `drawBrand` (gold nameplate — the toolbar icon matches it). |
| `winamp.css` | Procedural chrome + GEN/PLEDIT frames. |
| `icons/` | **Gold "N" toolbar icon (new)** — matches the nameplate (`#fff7dc→#f0d182→#a87f28` on a `#d8b863→#523e13` plaque). Generated by a Node PNG encoder (see git history / scratchpad). |
| `manifest.json` | + `tabCapture`, `offscreen`, `activeTab`, `contextMenus` perms; `action` (gold-N icon, no popup); `background.service_worker`; `commands` (`Ctrl/Cmd+Shift+E`). |

## 2. How to launch + how to verify

**Launch (one click, NO share popup):** click the gold **N** toolbar icon, OR right-click
the page → "Toggle NeoAmp player + EQ", OR `⌘⇧E`. Any of these *invokes the extension*,
which is what `tabCapture` requires — an in-page button legally cannot start capture.
`tabCapture` shows **no picker dialog** (unlike the old getDisplayMedia). On Arc, the
toolbar icon is hidden by default — pin it: `⌘⇧D` to show the toolbar, then Site Control
Center → 📌 next to NeoAmp.

**Headless render harness** (UI only — it mocks `window.NeoAmp`, so it can't exercise
tabCapture/offscreen/audio):

```bash
node tools/render-neoamp.mjs                 # default stack + library/search/home/eq → /tmp/neoamp-render/*.png
SKIN=TopazAmp node tools/render-neoamp.mjs
```

The **audio + EQ + visualizer can't be headless-tested** (no audio in headless Chrome) —
they need a live reload + ears/eyes. Iterate UI/layout with the harness; confirm audio live.

## 3. The real-EQ architecture (the big rebuild, 2026-06-19)

Goal: an EQ that shapes **what you hear**. YTM audio is cross-origin (can't
`createMediaElementSource`), so we must capture + replay a processed copy.

```
right-click / gold-N icon (user gesture, "invokes" extension)
  → sw.js toggleCapture: chrome.tabCapture.getMediaStreamId({targetTabId})  → streamId
  → chrome.offscreen.createDocument(reasons:['USER_MEDIA'])
  → offscreen: getUserMedia(tab streamId) → AudioContext
       src → preamp → [10 biquads] → balance → master → destination   (AUDIBLE, EQ'd)
                                  ↘ analyser → setInterval(20ms) → base64 → sw → content → viz
  → sw lifecycle 'started' → content emits "start" → winamp showUI (player appears)
EQ window fader → NA.control.setEqBand → content relay-eq → sw → offscreen setEq (live)
```

tabCapture **mutes the tab's own output**; replaying through the offscreen
AudioContext's `destination` is what keeps it audible (and EQ'd). EQ state persists to
`chrome.storage` (`neoampEq`); the offscreen reads it on start + on live `setEq` messages.

## 5. Gotchas worth keeping (hard-won this session)

- **tabCapture needs the extension "invoked"** (toolbar icon / context menu / command /
  keyboard shortcut) + a fresh user gesture in the SW context. A content-script button
  relaying a message does NOT count — error: *"Extension has not been invoked for the
  current page (see activeTab permission)."* So START can only come from those; STOP can
  come from anywhere (no gesture needed).
- **`suppressLocalAudioPlayback:true` mutes the WHOLE tab's local output — including your
  own WebAudio `destination`** (same tab). That's why the getDisplayMedia approach to a
  real EQ failed (silence). tabCapture + a *separate* (offscreen) context is the only fix.
- **`requestAnimationFrame` does NOT fire in an offscreen document** (no rendering). Use
  `setInterval` for the FFT pump (this exact bug froze the visualizer).
- **tabCapture shows NO share/picker popup** (unlike getDisplayMedia). One click = capture.
- **Runtime/tabs messaging is JSON** (no typed arrays) → FFT is base64-packed
  `[time(1024)|freq(512)]` and unpacked in content.js.
- **viz.html is a sandboxed iframe = opaque origin** → can't use BroadcastChannel with the
  offscreen. FFT must route offscreen→SW→content→`postMessage`→viz (the existing bridge).
- **getDisplayMedia and tabCapture can't both capture one tab** — don't run both.
- **CDP `Runtime.evaluate`** returns at `result.result.value`; return `JSON.stringify` for objects.
- **Framed (GEN/PLEDIT) titlebar must be `flex: 0 0 40px`**; the close glyph is baked into
  the corner sprite (overlay a transparent hit-area, never draw a second ✕).
